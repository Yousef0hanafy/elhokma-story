# رحلة الحوكمة — Premium Arabic Story-Based E-Learning Prototype

> النموذج الأولي — مشهدان من تجربة تعلّم سينمائية تفاعلية بالعربية الفصحى

---

## المفهوم الإبداعي

تحوّل المتعلم إلى **عضو جديد في مجلس إدارة مستشفى السلام التخصصي**. ترافقه throughout the journey **د. سارة الراشد** — مستشارة الحوكمة — كمرشد سردي يفتح له الأبواب ويقدم السياق قبل كل مفهوم.

**الفلسفة التصميمية:**
- لا نوّر المفاهيم — نوّر الشخصيات. كل مفهوم حوكمة يتحوّل إلى "شخصية" في القصة.
- السرد قبل المفهوم. التفاعل قبل التقييم.
- الحركة تخدم التعلم — كل تأثير حركي له سبب.
- العربية الفصحى كلغة أصلية، لا ترجمة.

## المشهدان المُنتَجان

### المشهد ١ — الافتتاح: اليوم الأول
- **النوع:** مقدمة سينمائية (Cinematic Intro)
- **المدة:** ~31 ثانية سرد + 3 ثوانٍ ظهور الزر
- **العناصر:**
  - شاشة سوداء ← ظهور تدريجي
  - أشرطة سينما (Letterbox) تنزلق من الأعلى والأسفل
  - عنوان "رحلة الحوكمة" يُكتب بتأثير الآلة الكاتبة
  - ظهور الراوية د. سارة على اليمين مع توهّج ناعم
  - ٩ مقاطع سرد موقوتة تظهر كترجمة في الأسفل
  - مؤشّر موجة صوتية ينبض بجانب الراوية أثناء الكلام
  - زر "ابدأ الرحلة" يتجسّد بتوهّج ذهبي ونبض لطيف

### المشهد ٢ — غرفة القرار (استكشاف موجّه)
- **النوع:** Guided Exploration
- **المدة:** ~37 ثانية سرد + استكشاف ذاتي
- **العناصر:**
  - طاولة بيضاوية مرسومة بأسلوب stroke-draw (رسم الخط تدريجياً)
  - ٦ مقاعد حول الطاولة، كل واحد يمثّل مكوّن حوكمة
  - أثناء السرد، يتوهّج كل مقعد عند ذكره
  - بعد السرد، تصبح المقاعد قابلة للنقر
  - كل نقرة تفتح بطاقة منبثقة بقصة سرديّة + تعريف رسمي
  - شريط تقدّم: "استكشفت X من ٦"
  - بعد استكشاف الستة: يظهر بند تقييم مع تغذية راجعة غنية

## لافتات إبداعية

### ١. لوحة "ملاحظات الإنتاج" (Director's Commentary Mode)
زر "🎬 ملاحظات الإنتاج" في الشريط العلوي يفتح لوحة جانبية تعرض **المواصفات الـ15 كاملة للمشهد الحالي** — العنوان، الهدف، الغاية السردية، نص السرد، الخط الزمني للحركة، التوجيه البصري، نوع التفاعل، منطق التفاعل، بند التقييم، التغذية الراجعة، ملاحظات التنفيذ، توافق SCORM، هيكل JSON، ومعمارية الكود. هذا تحويل لمفهوم "مستند التصميم" إلى تجربة تفاعلية داخل المنتج نفسه.

### ٢. تأثير الآلة الكاتبة للترجمة
بدلاً من ظهور السطر دفعة واحدة، يُكتب حرفاً حرفاً بسرعة متكيّفة (20-45 مللي ثانية/حرف) تتماشى مع طول النص. يُظهر مؤشّر ذهبي ينبض في النهاية، ثم يختفي بعد 600 مللي ثانية. في وضع `prefers-reduced-motion`، يظهر النص كاملاً فوراً.

### ٣. الراوية كـ SVG inline
د. سارة مرسومة كـ SVG مسطّح باحترافية (وجه، حجاب، معطف طبي، أقراط ذهبية) — لا اعتماد على صور خارجية، جاهزة للعمل في بيئات SCORM المقيدة. تظهر بمقياس كامل في المشهد الأول، ثم مصغّرة (90px) كمرشد صامت في المشهد الثاني.

### ٤. التوهّج الموقوت للمقاعد
أثناء السرد، يتوهّج كل مقعد لمدة 3.5 ثوانٍ عند ذكر دوره. هذا يربط الكلمة المنطوقة بالعنصر البصري، مما يعزّز الاستيعاب عبر قناتين معرفيتين (سمعية + بصرية).

### ٥. لوحة ألوان Dark Editorial
أزرق ليلي عميق (#0B1F33) + ذهبي (#D4AF37) + كريمي (#F4ECD8) — تذكّر بأفلام الإثارة السياسية والمستشفيات الليلية، لا بقوالب PowerPoint التقليدية.

## التقنيات

| التقنية | الاستخدام |
|---------|-----------|
| Vanilla JS (no framework) | إمكانية التشغيل في بيئات SCORM المقيدة بـ CSP |
| SVG inline | الطاولة، المقاعد، صورة الراوية — لا ملفات خارجية |
| CSS Custom Properties | لوحة ألوان موحّدة قابلة للتعديل من مكان واحد |
| Google Fonts (Tajawal + Cairo + Amiri) | طباعة عربية احترافية بثلاثة أنماط (Modern + Body + Editorial) |
| requestAnimationFrame | حركات سلسة دون jank |
| Web Animations API patterns | عبر `setTimeout` chains للتحكم الكامل في التخطّي |
| LocalStorage fallback | وضع مستقل عند التشغيل خارج LMS |

## توافق SCORM

| الحقل | القيمة |
|-------|--------|
| `cmi.core.lesson_status` | `not attempted` → `incomplete` → `completed` / `passed` |
| `cmi.core.score.raw` | 50 (النموذج الأولي يغطّي نصف المشاهد) |
| `cmi.core.score.min` / `max` | 0 / 100 |
| `cmi.core.session_time` | HH:MM:SS (تحديث كل 60 ثانية + عند الإغلاق) |
| `cmi.suspend_data` | JSON يحوي: المشهد الحالي، المقاعد المُستكشَفة، إجابة التقييم، حالة السرد |
| `cmi.core.student_name` | يُقرأ للترحيب (في LMS الحقيقي) |

**وضع مستقل:** عند التشغيل خارج LMS، يحفظ التقدّم في `localStorage` تلقائياً. افتح `index.html` مباشرة في المتصفح لتجربة النموذج.

## قابلية التوسّع

إضافة مشهد ثالث = خطوتان فقط:
1. أضِف كائناً جديداً إلى `STORY_CONTENT.screens[]` في `js/content.js` (مع المواصفات الـ15 الكاملة).
2. أضِف دالة `render<SceneName>(scene)` في `js/app.js` واربطها في `renderScene()`.

كل البنية التحتية (السرد، الحركة، الترجمة، SCORM، اللوحة الجانبية) تعمل تلقائياً مع أي مشهد جديد.

## كيفية المعاينة

### الطريقة ١: معاينة محلية مباشرة
افتح `index.html` مباشرة في أي متصفح حديث. يعمل فوراً دون أي إعداد.

### الطريقة ٢: عبر خادم محلي
```bash
cd lms_story && python3 -m http.server 8000
# افتح http://localhost:8000
```

### الطريقة ٣: الرفع إلى LMS
ارفع المجلد كم SCO عبر SCORM 1.2 — لم يُضمَّن `imsmanifest.xml` في النموذج الأولي عمداً لتسهيل المعاينة. سيُضاف عند الانتقال إلى الإنتاج الكامل.

## ما الذي ننتظر رؤيته منك؟

قبل التوسّع إلى المشاهد الـ٤ المتبقية، نودّ رأيك في:

1. **الأسلوب السردي:** هل د. سارة الراشد كمرشد تعمل؟ أم تفضّل راوياً محايداً؟
2. **إيقاع الحركة:** الـ31 ثانية للمقدمة — هل هي طويلة؟ قصيرة؟
3. **لوحة الألوان:** Dark Editorial — هل تناسب السياق الصحي السعودي؟ أم تفضّل لوحة فاتحة؟
4. **Guided Exploration:** هل فكرة "غرفة القرار" بالأمكنة الستة تعمل؟ أم تفضّل عرضاً أكثر تقليدية؟
5. **لوحة ملاحظات الإنتاج:** هل هي مفيدة لمراجعة المواصفات؟ أم تشتّت؟
6. **نبرة التغذية الراجعة:** "لِنُراجِع..." — هل هي محترمة أم تقريرية؟
7. **المح移动:** هل التوهّج الموقّت أثناء السرد يضيف أم يشتّت؟

أيّ تعليق على هذه المحاور يُمكن أن يُعيد تشكيل المشاهد اللاحقة بشكل جوهري قبل أن نبنيها.
