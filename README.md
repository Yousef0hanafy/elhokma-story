# رحلة الحوكمة — Premium Arabic Story-Based E-Learning

> تجربة تعلّم سينمائية تفاعلية بالعربية الفصحى عن الحوكمة في القطاع الصحي

---

## نظرة عامة

رحلة الحوكمة هي تجربة تعلّم ذاتية الإيقاع تأخذ المتعلم في رحلة سردية من اليوم الأول في مجلس إدارة مستشفى السلام التخصصي. يرافقه خلال الرحلة **د. سارة الراشد** — مستشارة الحوكمة — كمرشد افتراضي يفتح له الأبواب ويقدّم السياق قبل كل مفهوم.

### الفلسفة التصميمية

- **السرد قبل المفهوم.** كل مفهوم يُقدَّم عبر قصة قبل أن يُعرَّف.
- **الحركة تخدم التعلم.** كل تأثير حركي له سبب تعليمي.
- **العربية الفصحى كلغة أصلية**، لا ترجمة.
- **الاستكشاف قبل التقييم.** المتعلم يستكشف أولاً، ثم يُختبر.

## المزايا الرئيسية

### 🎙️ السرد الصوتي العربي
- استخدام Web Speech API للتشغيل التلقائي للسرد العربي
- كتم/تشغيل + تحكم بالسرعة (٠.٧٥× حتى ١.٥×)
- مزامنة تلقائية مع الترجمة النصية
- يعمل دون اتصال إنترنت (بعد تحميل الصفحة)
- ترجمة نصية دائماً متاحة كـ fallback

### 🎬 الحركة السينمائية
- أشرطة سينما (Letterbox) للإطار الدرامي
- تأثير الآلة الكاتبة للعناوين والترجمة
- توهّج موقّت للعناصر أثناء السرد
- انتقالات متلاشية بين المشاهد
- خلفية محيطة بحركة خفيفة

### 🧭 التنقل المرن
- **الوضع الموجّه:** السرد الكامل مع التفاعل التدريجي
- **الوضع الذاتي:** قائمة المشاهد تتيح القفز لأي مشهد مفتوح
- أزرار: تخطّي السرد (»)، إعادة السرد (↺)
- زر "إعادة الرحلة من البداية" في قائمة المشاهد

### 🎨 التصميم البصري
- لوحة Dark Editorial: أزرق ليلي (#0B1F33) + ذهبي (#D4AF37) + كريمي (#F4ECD8)
- ثلاثة خطوط عربية: Tajawal (عناوين) + Cairo (نصوص) + Amiri (تحريري)
- راوية مرسومة كـ SVG inline (د. سارة بعباءة طبية وحجاب أنيق)
- متجاوب بالكامل: 390px → 1400px+

### ♿ إمكانية الوصول
- دعم قارئات الشاشة (ARIA labels)
- التنقل بلوحة المفاتيح
- دعم `prefers-reduced-motion`
- تباين لوني WCAG AA
- أهداف لمس 44px كحد أدنى

## البنية التقنية

```
lms_story/
├── index.html              ← نقطة الدخول
├── css/
│   ├── style.css           ← التصميم الأساسي + التنقل + TTS
│   ├── animations.css      ← مكتبة الحركات
│   └── scenes.css          ← أنماط المشاهد ٣-٧
├── js/
│   ├── scorm-api.js        ← غلاف SCORM 1.2 + localStorage fallback
│   ├── tts.js              ← محرّك السرد الصوتي (Web Speech API)
│   ├── content.js          ← المحتوى + مواصفات الإنتاج الـ15 لكل مشهد
│   ├── narrator.js         ← محرّك السرد (ترجمة + كاتب + مزامنة)
│   ├── animator.js         ← منسّق الحركة
│   └── app.js              ← التطبيق الرئيسي + ٧ دوال عرض
├── preview/                ← لقطات شاشة مرجعية
└── README.md
```

### التقنيات
- **Vanilla JS** (بلا أطر عمل) — متوافق مع بيئات SCORM المقيدة
- **SVG inline** — الراوية، طاولة المجلس، خريطة النجوم
- **Web Speech API** — السرد الصوتي العربي
- **CSS Custom Properties** — لوحة ألوان موحّدة
- **Google Fonts** — Tajawal + Cairo + Amiri

## توافق SCORM

| الحقل | القيمة |
|-------|--------|
| `cmi.core.lesson_status` | `not attempted` → `incomplete` → `completed` / `passed` |
| `cmi.core.score.raw` | 0–100 (تقدّمي: كل مشهد = ١٠٠/٧ ≈ ١٤٪) |
| `cmi.core.session_time` | HH:MM:SS (تحديث كل 60 ثانية) |
| `cmi.suspend_data` | JSON يحوي: المشهد الحالي، درجات المشاهد، حالة كل مشهد |

**وضع مستقل:** عند التشغيل خارج LMS، يحفظ التقدّم في `localStorage`.

## كيفية المعاينة

### معاينة محلية
افتح `index.html` مباشرة في أي متصفح حديث. يعمل فوراً.

### وضع المطور (ملاحظات الإنتاج)
أضِف `?dev=1` إلى الرابط لعرض زر "🎬 ملاحظات الإنتاج" الذي يفتح لوحة بكل مواصفات التصميم الـ15 للمشهد الحالي.

```
index.html?dev=1
```

## التجربة المثلى

لأفضل تجربة تعلّم:
1. استخدم سماعات الرأس للاستمتاع بالسرد الصوتي
2. اسمح بتشغيل الصوت عند سؤال المتصفح
3. اتبع الوضع الموجّه في المرة الأولى
4. استخدم قائمة المشاهد للمراجعة لاحقاً
5. جرّب على الجوال للتأكد من التجاوب الكامل
